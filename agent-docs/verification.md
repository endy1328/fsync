# 검증 가이드

## 목적

이 문서는 이 저장소에서 완료된 작업을 어떤 기준으로 검토할지 정의합니다.

## 검증 대상

- 문서 작업
- 코드 작업
- 다중 에이전트 통합 작업

## 기본 검토 체크리스트

- 결과가 사용자 요청과 일치하는가
- 요청 범위 밖 작업이 불필요하게 추가되지 않았는가
- 관련 문서가 함께 갱신되었는가
- 책임과 소유 범위가 명확히 분리되었는가
- 문서 간 용어와 현재 단계 표기가 일관적인가
- 파일 간 명백한 충돌이 없는가
- 숨은 가정 없이 다음 단계로 진행 가능한가
- 핵심 sync 동작이 자동 테스트로 덮여 있는가

## 구현 검토 중점

- TOML `[[jobs]]` 설정이 정상 로드되는가
- `run`, `once` 동작이 CLI 계약과 호환되는가
- 신규 및 수정 파일이 모든 target으로 복사되는가
- 재시작 후에도 변경되지 않은 파일을 다시 복사하지 않는가
- `copy_deleted=true`일 때 target 삭제가 반영되는가
- busy source 파일이 보류되고 다음 주기에 재시도되는가
- 여러 target 중 일부만 오래되었을 때 필요한 target만 복사되는가
- target 쓰기/삭제 실패가 현재 job 실패로 처리되는가
- target metadata 반영 실패가 target 측 실패로 처리되는가
- job 단위 병렬 실행이 다른 job을 막지 않는가
- `run_once`가 모든 job 종료 후 집계된 실패를 보고하는가

## 코드 리뷰 중점

- 요구사항 충족 여부
- 회귀 위험
- 누락된 테스트
- 실행 또는 설정 위험

## 현재 검증 명령

- `python3 -m unittest discover -s tests -v`

## 현재 합격 기준

- 핵심 sync 시나리오가 단위 테스트를 통과해야 합니다
- busy source 파일은 현재 실행에서 건너뛰고 다음 실행에서 재시도해야 합니다
- 변경되지 않은 파일은 재시작 후에도 다시 복사되면 안 됩니다
- one-shot 실행에서 한 job 실패가 다른 job을 조기 중단시키면 안 됩니다
- README와 운영 문서가 구현 상태와 일치해야 합니다

## 검토 결과 형식

- 검토 대상
- 수행한 확인 항목
- 발견된 이슈
- 통과 여부
- 후속 조치

## 재검증 트리거

- 검증 이슈를 수정했을 때
- 작업 중 범위가 바뀌었을 때
- 검토 후 관련 문서가 변경되었을 때
- 병렬 작업 결과를 메인 에이전트가 통합했을 때
