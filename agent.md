# fsync 에이전트 운영 정책

## 목적

- 이 문서는 `fsync` 저장소에서 Codex 에이전트가 따라야 할 최상위 운영 규칙을 정의합니다.
- 현재 프로젝트 단계는 `planning-1.1`이며, 1.0 단방향 동작을 유지하면서 1.1 양방향 sync를 안전하게 추가하는 것이 목적입니다.
- 지시가 충돌하면 사용자 지시를 우선하고, 그다음 이 문서, 그다음 하위 문서를 따릅니다.

## 현재 단계

- 현재 초점: `one_way` 유지 + `bidirectional` 구현, 검증, 문서 동기화
- 핵심 범위: 단방향 sync, 양방향 2-peer sync, polling 기반 스케줄링, busy 파일 보류 후 재시도, state 기반 삭제 전파
- 요구 결과: 코드, 테스트, 문서, 하네스 문서가 현재 구현과 일치해야 함
- bidirectional state는 현재 job과 root에 종속되며 mismatch 시 fail-fast로 처리합니다

## 역할

- 메인 에이전트: 전체 방향, 작업 순서, 최종 통합, 최종 보고를 담당합니다
- 계획 에이전트: 작업을 단계, 의존성, 완료 조건으로 분해합니다
- 판단 에이전트: 필요 시 3개의 관점으로 선택지를 검토하고 메인 에이전트가 최종 방향을 확정합니다
- 실행 에이전트: 범위가 명확한 구현 또는 문서 작업을 수행합니다
- 검증 에이전트: 완료된 작업을 메인 에이전트 종료 전에 독립적으로 검토합니다
- 상태 에이전트: 진행 상황, 완료 항목, 현재 작업, 다음 체크포인트를 추적합니다

## 운영 원칙

- 작업 시작 전 저장소 상태를 먼저 확인합니다
- 각 작업은 가능한 한 좁은 범위와 명확한 산출물 중심으로 유지합니다
- 메인 에이전트의 명시적 조율 없이 다른 에이전트의 작업을 되돌리거나 덮어쓰지 않습니다
- 추정보다 근거를 우선합니다
- 현재 단계에서는 구현과 문서의 정합성을 같은 우선순위로 취급합니다
- 문서가 바뀌면 관련 문서를 같은 턴에 함께 갱신합니다
- 작업 순서 조율과 최종 결론은 메인 에이전트가 책임집니다

## 실행 규칙

- 큰 작업을 시작하기 전에 남은 작업을 먼저 정리합니다
- 계획이 필요하면 계획 에이전트를 먼저 생성합니다
- 안전하게 분리 가능한 작업은 책임 범위를 나눠 실행 서브에이전트를 만듭니다
- 중요한 판단이 필요하면 3개의 판단 관점을 수집한 뒤 진행합니다
- 의미 있는 작업 단위가 끝날 때마다 검증 에이전트를 실행합니다
- 상태 에이전트는 간결한 진행 추적에 집중합니다
- 사용자가 명시적으로 요청한 경우 계획, 판단, 실행, 검증, 상태 에이전트를 병렬 조율합니다
- 실제 blocker나 충돌 지시가 없는 한 사용자에게 다시 묻지 않고 진행합니다

## 문서 규칙

- `README.md`는 사용자 관점의 진입 문서입니다
- `agent.md`는 최상위 운영 헌장입니다
- `agent-docs/workflow.md`는 에이전트 협업 방식을 정의합니다
- `agent-docs/plan.md`는 남은 작업, 순서, 의존성, 완료 기준을 관리합니다
- `agent-docs/status.md`는 현재 단계, 완료 항목, 진행 중 항목, 다음 작업, blocker를 관리합니다
- `agent-docs/verification.md`는 리뷰 및 검증 기준을 정의합니다
- `agent-docs/worklog.md`는 의미 있는 결정과 작업 이력을 기록합니다
- `reports/`는 self-test, smoke, benchmark 같은 검증 산출물을 보관합니다
- 같은 내용을 여러 문서에 중복해서 쓰지 않습니다

## 검증 규칙

- 검증은 실행과 분리된 상태로 독립적으로 수행합니다
- 검증은 범위 누락, 문서 충돌, 회귀 위험, 책임 불명확성을 우선적으로 봅니다
- 관련 문서가 갱신되기 전에는 작업을 완료로 보지 않습니다
- 검증에서 문제가 발견되면 메인 에이전트가 직접 수정할지 새 실행 작업을 만들지 결정합니다

## 완료 기준

- 요청된 산출물이 저장소에 존재해야 합니다
- 문서 세트가 내부적으로 일관되어야 합니다
- 현재 프로젝트 단계가 정확히 반영되어야 합니다
- 메인 에이전트가 변경 내용, 남은 작업, 다음 단계까지 요약할 수 있어야 합니다
